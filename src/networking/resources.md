# Class Resources

Here are some of the resources that might be useful for the class:

__Networking:__
- __[https://intronetworks.cs.luc.edu/current2/html/](https://intronetworks.cs.luc.edu/current2/html/)__
    - Not the official textbook, but is recommended by the professor as supplemental material
- [https://netsim.erinn.io/](https://netsim.erinn.io/)
    - Gives a good visual representation of small networks and lets you create/simulate your own packets
- [https://developer.mozilla.org/en-US/docs/Learn_web_development/Howto/Web_mechanics/How_does_the_Internet_work](https://developer.mozilla.org/en-US/docs/Learn_web_development/Howto/Web_mechanics/How_does_the_Internet_work)
    - A good primer for how the Internet works + the purposes of network hardware
- [https://potatopirates.game/pages/introduction-to-computer-networking](https://potatopirates.game/pages/introduction-to-computer-networking)
    - Asks you to sign up for a newsletter, but gives an overview of networking concepts with cool illustrations

__Intro to Linux + using a terminal:__
- [https://ryanstutorials.net/linuxtutorial/](https://ryanstutorials.net/linuxtutorial/)
    - Explains linux, while including a lot of useful details for beginners that are usually left unsaid
- [https://developer.mozilla.org/en-US/docs/Learn_web_development/Getting_started/Environment_setup/Command_line](https://developer.mozilla.org/en-US/docs/Learn_web_development/Getting_started/Environment_setup/Command_line)
    - Just like the linux tutorial above, but shorter/more general (includes details about Windows/Mac terminals)
- [https://primer.picoctf.org/#_got_shell](https://primer.picoctf.org/#_got_shell)
    - While it's aimed for beginning competitive hackers, this section + the following ones give a great introduction to the command line and common acronyms.
    - You can ignore the instructions about signing up for a CTF, since we'll create our own Linux VM in the first week
- [https://explainshell.com/about](https://explainshell.com/about)
    - explains linux commands (line by line if they're complicated), basically a more specialized google
